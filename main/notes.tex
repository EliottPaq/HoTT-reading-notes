\section{Type Theory}
\subsection{Type theory versus set theory}

Alternative à ZF, Set theory à deux couches: logique du premier ordre et les systèmes d'axiomes, permet d'analyser les interactions entre set. Contrairement Type theory est son propre système déductif \(\to\) pas besoin de superstructure. Une notion basique \textit{types}.\\
Les preuves sont confondues avec construction d'objet (facilement traduit en info ?). Dans Set theory une proposition peut être démontré ou réfuter, dans type theory, la preuve est l'appartenance à un type, et ça n'as pas de sens de considérer cette proposition comme fausse ou pas, car \(a : A\) est atomique. \\
Différence dans l'égalité : Set theory \( \to \) relation entre deux objets (vrai ou pas) type theory : \(\to \) deux types d'égalité, l'égalité propositionnelle si \(a,b : A\) alors, il existe un type \(a =_{A} b\) et l'égalité de définition noté \(a \equiv b\)
\subsection{Function Types}
Utilisation du lambda calcul dans la représentation des fonctions \(\paren{\lambda \paren{x:A}.\phi}: A \to B\)
Curryfication des fonctions (prénom de Curry  : Haskell !)
\subsection{Universes and families}

Univers est un type où ces éléments sont des types. Pas d'univers de tous les types à cause du paradoxe de Russell. On peut créer une hiérarchie d'univers \(\mathcal{U}_0:\cal{U}_1:\cal{U}_3:\dots\).
Pour modéliser une collection de type variant sur un type \(A\) on utilise la fonction \(B : A \to \cal{U}\) ces fonctions sont appelées famille de type.
\subsection{Dependent function type (\(\Pi\)-types)}

Les éléments d'un \(\Pi\)-types sont des fonctions dont le type codomaine peut varier en fonction de l'élément du domaine duquel la fonction est appliqué. Exemple : Si \(A:\cal{U}\) et une famille \(B : A \to \cal{U}\) on peut construire une fonction type-dépendant \(\Pi_{\paren{x:A}} B(x) : \cal{U}\) \\
Principe d'unicité : \(f \equiv \paren{\lambda x.f(x)}\) pour tout \(f : \Pi_{x:A}B(x)\). \\
On peut définir des fonctions polymorphiques comme l'identité telle que \(id : \equiv \lambda\paren{A : \cal{U}}.\lambda \paren{x : A}.x\) ou encore swap : \[\fct{swap} : \Pi_{A:\cal{U}}\Pi_{B:\cal{U}}\Pi_{C:\cal{U}} \paren{A \to B \to C} \to \paren{B \to A \to C}\] définie par :\[\fct{swap}(A,B,C,g):= \lambda b . \lambda a.g(a)(b)\] 
\subsection{Product Types}
En set theory, quand on définit une fonction sur \(A \times B\), on suppose que les couples sont bien ordonnés, or en type theory, on suppose que la fonction est bien définie, ce qui permet de démontrer que les couples sont des paires.\\
Pour définir les projecteurs, on peut passer par le récurseur pour les types produits définit par le type \[\fct{rec}_{A \times B} : \Pi_{C :\cal{U}}\paren{A \to B \to C} \to A \times B \to C\] avec la définition \[\fct{rec}_{A \times B} \paren{C,g,(a,b)}: \equiv g(a)(b)\] ce qui permet de dire \[pr_1 : \equiv \fct{rec}_{A\times B}(A,\lambda a.\lambda b.a)\]
\[pr_2 : \equiv \fct{rec}_{A \times B} (B,\lambda a. \lambda b.b)\]

On peut définir le récurseur à partir des projections :
\[\fct{rec}_{A\times B} (C,g,d) : \equiv g(pr_1(d))(pr_2(d))\]
Et il est évident que cette définition du récurseur est bien dérivé de cette définition. \\
Si on a \[g : \Pi_{x:A}\Pi_{y:B}C((x,y))\] alors on peut définir \(f : \Pi_{x:A \times B} C(x)\) par \[f((x,y)): \equiv g(x)(y)\]
Ce qui est utile pour prouver le principe d'unicité propositionnelle qui montre que tout élément de \(A \times B\) est égal à une paire.
Via \[\fct{uniq}_{A \times B} : \Pi_{x : A \times B} \paren{\paren{pr_1 (x),pr_2(x)} =_{A \times B} x}\]
Lorsque le type seras introduit sous-section 12 on pourra utiliser l'élément réflexif \(\fct{refl}_x : x =_{A} x\) pour tout \(x:A\) pour définir \[\fct{uniq}_{A \times B}: \equiv \fct{refl}{(a,b)}\]
On peut aussi créer la fonction induction pour les types produits : \[\fct{ind}_{A \times B}  : \Pi_{C:A \times B \to \cal{U}}\paren{\Pi_{\paren{x:A}}\Pi_{\paren{y:B}}C((x,y))}\to\Pi_{x:A\times B}C(x)\]
\[\fct{ind}_{A \times B} \paren{C,g,(a,b)}:\equiv g(a)(b)\]
